You’re asking an excellent (and difficult) question. My mobile days are long gone, but the alternative seems to be:

  1. write the same thing three times. Yes, it’s painful, but at least, you get to write the best code for each platform.
  2. write portable pre-processing code. I don’t think the training environment is a problem here, as you can literally run any language on AWS, and specifically on SageMaker with a custom container. The old fart in me would point you at C / C++, which can run on both mobile platforms. APIs are obviously different, but at least you should be able to get some code reuse… Now, if you also need extra native libraries for pre-processing (image, etc.), this might add extra portability issues.
  3. Offload pre-processing and prediction to a cloud-based API. It all depends on whether you need connectivity at all times, on how much data you have to work with, and on how much latency you can take.

I’m sure you’ve considered all three already. If I had to choose? 3. If unfeasible, 1. Portability is much harder than most people think.

Hope this helps.
